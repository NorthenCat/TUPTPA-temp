TUPTPA Atomic Design

Penjelasan struktur Atomic Design berdasarkan komponen real di project

Dokumentasi ini menjelaskan bagaimana routing, template, organism, molecule, dan atom bekerja di project Angular TUPTPA. Contoh yang dipakai diambil dari file yang sudah ada di project.

Alur Besar

Routing memilih layout utama. Setiap file page juga dibungkus dengan app-page-template agar pemanggilan atomic terlihat langsung dari halaman. Dari sana template menyusun organism, organism menyusun molecule, dan molecule memakai atom sebagai komponen paling kecil.

Routing Menentukan layout dan page yang aktif.
Template Struktur besar halaman seperti layout admin.
Organism Bagian besar seperti navbar, sidebar, configuration.
Molecule Gabungan komponen kecil seperti card, modal, breadcrumb.
Atom Komponen terkecil seperti button, badge, alert, spinner.

1. Routing Memanggil Template

Pada project ini, routing memakai LayoutComponent sebagai template utama untuk route admin.

src/app/app-routing.module.ts
{
  path: '',
  component: LayoutComponent,
  canActivate: [AuthGuard],
  data: { template: 'admin' },
  children: [
    {
      path: 'home',
      loadChildren: './pages/_home/home.module#HomeModule'
    },
    {
      path: 'sample-page',
      loadChildren: './pages/sample-page/sample-page.module#SamplePageModule'
    }
  ]
}
Catatan: Routing bukan tempat untuk memanggil atom. Routing cukup memanggil template, lalu template yang akan menyusun komponen di bawahnya.

2. Template

Template adalah struktur besar halaman. Di project ini ada dua contoh penting: LayoutComponent sebagai app shell utama dan PageTemplateComponent sebagai wrapper yang dipanggil langsung oleh setiap page.

src/app/shared/templates/layout/layout.component.html
<ng-container *ngIf="isAdminTemplate; else blankTemplate">
  <app-navigation></app-navigation>
  <app-nav-bar></app-nav-bar>

  <div class="pcoded-main-container">
    <div class="pcoded-content">
      <app-breadcrumb></app-breadcrumb>
      <router-outlet></router-outlet>
    </div>
  </div>

  <app-configuration></app-configuration>
</ng-container>

Cara kerjanya: LayoutComponent hanya membungkus struktur halaman. Konten page sebenarnya akan masuk melalui <router-outlet>.

src/app/pages/*/*.component.html
<app-page-template>
  <!-- isi page asli -->
</app-page-template>
src/app/shared/templates/page-template/page-template.component.html
<app-page-content [loading]="loading">
  <ng-content></ng-content>
</app-page-content>

Dengan ini setiap halaman terlihat jelas memanggil template atomic, tanpa menggandakan sidebar atau navbar.

3. Organism

Organism adalah komponen besar yang menyusun beberapa bagian UI. Contoh real di project: app-nav-bar, app-navigation, dan app-configuration.

Organism

Navbar

src/app/shared/organisms/nav-bar/nav-bar.component.html
<div class="collapse navbar-collapse">
  <app-nav-left class="mr-auto"></app-nav-left>
  <app-nav-right class="ml-auto"></app-nav-right>
</div>

Navbar disebut organism karena menyusun beberapa bagian besar: kiri navbar, kanan navbar, search, user menu, dan action lain.

Organism

Navigation Sidebar

src/app/shared/organisms/navigation/navigation.component.html
<div class="navbar-wrapper">
  <app-nav-content></app-nav-content>
</div>

Sidebar disebut organism karena merupakan satu blok navigasi besar yang berisi group menu dan item menu.

4. Molecule

Molecule adalah gabungan beberapa atom atau elemen kecil yang membentuk komponen reusable. Contoh real di project ini adalah app-card.

src/app/shared/molecules/card/card.component.html
<div class="card">
  <div class="card-header">
    <h5>{{ cardTitle }}</h5>

    <app-button
      buttonType="button"
      buttonClass="btn dropdown-toggle btn-icon"
      ariaLabel="button-toggle"
      [dropdownToggle]="true">
      <i class="feather icon-more-horizontal"></i>
    </app-button>
  </div>

  <ng-content></ng-content>
</div>

app-card disebut molecule karena memiliki header, title, action button, dan slot konten. Di dalamnya, molecule memakai atom app-button.

Contoh pemakaian di page
<app-card cardTitle="Daftar Data NKI">
  <!-- isi tabel, filter, atau action page -->
</app-card>

5. Atom

Atom adalah komponen paling kecil. Ia tidak tahu konteks page besar, hanya fokus pada satu fungsi. Contoh paling penting di project ini adalah app-button.

src/app/shared/atoms/button/button.component.html
<button
  [attr.type]="buttonType"
  [ngClass]="classes"
  [disabled]="disabled"
  (click)="handleClick($event)">
  <ng-content></ng-content>
</button>

app-button membungkus native <button>. Jadi page, molecule, atau organism tidak perlu membuat native button berulang-ulang.

Contoh pemakaian atom di page
<app-button
  buttonType="button"
  buttonClass="btn btn-primary"
  (click)="openTambahTkp()">
  Tambah Target Kinerja Pegawai
</app-button>

Contoh Chain Lengkap Dari Project

Berikut contoh hubungan antar level atomic design yang benar berdasarkan struktur project.

app-routing.module.ts -> LayoutComponent [template] -> app-nav-bar [organism] -> app-nav-left / app-nav-right [organism kecil] -> nav-search [bagian navbar] -> app-button [atom]
page component -> app-page-template [template] -> app-page-content [organism] -> app-content-shell [molecule] -> app-spinner [atom] -> isi page
page content -> app-card [molecule] -> app-button [atom]

Kenapa Atom Tidak Dipanggil Dari Routing?

Karena routing hanya bertugas menentukan halaman atau layout mana yang aktif. Atomic design bekerja di level komponen. Jadi atom akan dipakai di dalam template, organism, molecule, atau langsung di page jika memang dibutuhkan.

Jawaban singkat untuk client: Routing memanggil layout utama. Setiap halaman juga memanggil page template. Template menyusun organism. Organism dan molecule memakai atom. Jadi atom tidak perlu dan tidak seharusnya dipanggil langsung dari routing.

Ringkasan Kategori

Template Layout besar halaman. Contoh: LayoutComponent.
Organism Bagian besar UI. Contoh: navbar, sidebar, configuration.
Molecule Komponen reusable gabungan. Contoh: card, modal, breadcrumb.
Atom Komponen terkecil. Contoh: button, badge, alert, spinner.
